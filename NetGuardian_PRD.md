# NetGuardian 智能网络安全系统 - 核心需求与开发阶段说明

## 1. 项目愿景与架构约束
本项目旨在开发一个基于“端云协同”与深度学习的零信任网络安全态势感知平台。系统通过分析网络流量包的“时空特征”进行多分类威胁检测，并结合 LLM 提供安全诊断。

**【技术栈与架构约束（请 AI 严格遵守）】**
- **AI 算法端:** Python, Pandas, PyTorch 或 Scikit-Learn。
- **后端服务端:** `FastAPI` + `SQLModel`。
  - **强制约束:** 必须采用标准的**三层架构 (3-Tier Architecture)**，即明确区分 `Router/API层` (处理HTTP/WS请求)、`Service层` (处理业务逻辑和模型调用)、`Model/model层` (处理SQLModel数据库交互)。
  - **数据库:** SQLite (开发期) 或 PostgreSQL，用于持久化存储“拦截日志”和“流量审计记录”。
- **前端展示端:** - **架构决策权交由 AI:** 如果大屏布局、ECharts-GL (3D地球) 和 WebSocket 通信可以通过原生的 HTML/CSS (Tailwind)/JS (原生三件套) 优雅且不臃肿地实现，请优先使用原生开发。如果状态管理和组件复用极其复杂，请自行决定使用 Vue3 或 React，并说明理由。
- **大模型接口:** 兼容 OpenAI SDK 格式的 API 接口。

## 2. 核心业务需求描述

### 2.1 流量特征分类引擎 (感知)
- **输入数据:** 包含网络流统计特征的 CSV 数据集（包含包长、时间间隔等数值特征，最后一列为具体攻击类型 Label，如 DDoS、PortScan、BENIGN 等）。
- **任务目标:** 完成数据清洗、特征归一化，并训练一个**多分类 (Multi-class)** 深度学习或机器学习模型。
- **输出交付:** 训练好的模型权重文件，以及一个可以接收单条流量 JSON 数据并返回预测类别及置信度 (Probability) 的预测函数。

### 2.2 决策与业务控制中枢 (后端)
- **API 接口:** 提供接收流量特征的分析接口。
- **柔性阻断漏斗 (核心业务逻辑):**
  - **自动阻断:** 当预测为恶意攻击且 `置信度 > 0.95`，生成“自动封禁”指令，并将记录写入数据库。
  - **人工审批 (HITL):** 当预测为恶意攻击且 `0.75 < 置信度 <= 0.95`，生成“需审批”指令，写入数据库，并等待前端确认。
  - **放行:** `置信度 <= 0.75` 或预测为正常流量，直接放行。
- **实时通信:** 提供 WebSocket 服务，将处理后的日志和系统状态实时广播给前端大屏。

### 2.3 态势感知全息大屏 (前端)
- **视觉风格:** 科技暗黑风，布局清晰。
- **左侧实时日志区:** 动态滚动显示 WebSocket 传来的流量判定记录。
- **中央 3D 威胁地图:** 使用 ECharts-GL 绘制 3D 地球。当发生“自动阻断”时，在地球上渲染攻击源到目的地的红色飞线动画。
- **右侧图表区:** 展示流量吞吐量趋势（折线图）和系统安全评分（仪表盘）。
- **交互功能:** 当收到后端的“人工审批”请求时，页面弹出审批卡片，包含【一键封禁】与【忽略】按钮，点击后回调后端 API 更新数据库状态。

### 2.4 LLM 智能诊断助手 (Net-Copilot)
- **业务场景:** 当发生攻击拦截或人工审批时，将攻击特征和判定结果组合成 Prompt 提交给 LLM API。
- **输出要求:** LLM 需以“资深网络安全专家”的口吻，返回一段 100 字左右的通俗分析报告和安全建议，并在前端卡片中展示。

## 3. 渐进式开发步骤 (Execution Plan)

> **给 AI 编程助手的提示：** 请阅读以上完整需求，不要一次性写出所有代码。我们将按以下 4 个里程碑逐步推进。我会对你下达“开始阶段 X”的指令，每次请自行规划必要的文件结构和命名。

### 阶段 1：数据跑通与模型训练 (Model Foundation)
- **目标:** 读取 CSV 数据，完成预处理，训练多分类模型，保存模型并编写一个供后端调用的预测接口。
- **你的任务:** 评估数据处理逻辑，编写训练脚本并给出模型评估指标（准确率、召回率等）。

### 阶段 2：构建后端基座与三层架构 (Backend Infrastructure)
- **目标:** 搭建 FastAPI + SQLModel 后端框架。
- **你的任务:** 1. 设计 SQLModel 数据表结构（例如：`TrafficLog`）。
  2. 建立标准的 Router -> Service -> Model 三层目录结构。
  3. 实现接收流量数据的 API，在 Service 层集成“阶段 1”的预测接口，并实现“柔性阻断漏斗”逻辑，最后存入数据库。

### 阶段 3：前端大屏与 WebSocket 联调 (Visualization & Real-time)
- **目标:** 实现酷炫的态势感知前端，并与后端打通实时通信。
- **你的任务:** 1. 决定使用原生前端还是框架，并搭建基础页面。
  2. 编写后端 WebSocket 广播逻辑。
  3. 前端对接 WebSocket，实现滚动日志、ECharts 3D 地球飞线以及“人工审批”交互卡片。

### 阶段 4：大模型接入与全链路打通 (LLM Integration)
- **目标:** 为阻断和审批事件添加 LLM 的自然语言解释。
- **你的任务:** 编写调用大模型 API 的 Service 模块，设计 System Prompt，并将生成的诊断报告随着 WebSocket 一起推送到前端审批卡片中。
